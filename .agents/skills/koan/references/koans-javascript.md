# Коаны о JavaScript и TypeScript — примеры стиля

Эти коаны — эталон голоса, структуры и тона. Читай перед генерацией.

## Коан №1 — «О пустоте аргумента»

Из свитков монастыря Мацуо-дэра, 1174 г. Комментарий к ECMA-262, глава о призвании функций.

Ученик пришёл к мастеру и написал на песке:

```js
function gate(a, b, c) {}
gate(1);
```

Ученик: «Мастер, я передал одно — но врата ждут троих. Будет ли ошибка?»

Мастер ударил его палкой.

Мастер: «Ворота не закрываются от того, что гостей пришло меньше. Пустые места уже заняты — тишиной.»

Ученик: «Значит b и c — это undefined?»

Мастер: «Нет. Это mu. Они не определены — но они есть.»

Ветер дует — три чаши на столе. Одна полна.

С чем ты уйдёшь: Если undefined — это отсутствие значения, то чем отличается аргумент, который не передали, от аргумента, которому явно передали undefined?

## Коан №2 — «О равенстве неравных»

Из обгоревшего пергамента храма Энряку-дзи, 1231 г. Приписывается монаху Рэндану, толкователю Абстрактных Операций.

```js
[] == ![]; // true
```

Ученик: «Мастер, пустой массив равен отрицанию пустого массива. Это... невозможно.»

Мастер: «Ты спросил у языка "равны ли они?". Язык начал работу по Абстрактному Алгоритму Сравнения. Он не обязан думать о смысле.»

Ученик: «Но [] — это объект. ![] — это false. Как они могут быть равны?»

Мастер: «Потому что ты написал ==. Не ===. Ты попросил сравнения с принуждением. Язык принудил. Не жалуйся на результат принуждения, которое сам заказал.»

Ученик: «Тогда === всегда лучше?»

Мастер закрыл глаза.

Мастер: «Всегда — слово самонадеянных. Но да.»

Пустая чаша. Отрицание пустой чаши. Оба — на столе.

С чем ты уйдёшь: Язык исполняет правила, не намерения. Когда ты пишешь ==, ты подписываешь договор с Абстрактным Алгоритмом. Ты читал его?

## Коан №6 — «О порядке всех вещей»

Из запечатанного ларца в храме Кинкаку-дзи, 1378 г. Мастер Дзёсэцу продиктовал ученику перед смертью. Ученик ослушался и добавил setTimeout.

```js
console.log('1');
setTimeout(() => console.log('2'), 0);
console.log('3');
```

Ученик: «Мастер, я поставил ноль миллисекунд. Ноль! Это мгновение. Почему 2 приходит последним?»

Мастер налил чай в три чашки. Первую и третью подал сразу. Вторую поставил на поднос и передал слуге. Слуга обошёл комнату и вернул чашку на стол.

Ученик: «Но зачем?! Она была готова сразу!»

Мастер: «Ноль — это не "сейчас". Ноль — это "после всех, кто уже здесь". Ты попросил мгновение, но отправил просьбу через слугу. Слуга не войдёт, пока хозяин не замолчит.»

Ученик: «Event Loop — это слуга?»

Мастер допил чай.

Мастер: «Call stack — это хозяин. Пока он говорит — никто не входит. Ни промис. Ни таймер. Ни крик пользователя.»

Хозяин молчит. Слуга входит. Чашка — на столе.

С чем ты уйдёшь: setTimeout(fn, 0) — не "запустить сейчас". Это "запустить, когда освободится хозяин". Если хозяин никогда не замолкает — слуга не войдёт никогда.

## Коан №11 — «О наследовании без предков»

Фрагмент найден в дупле тысячелетней криптомерии на горе Хиэй, 1521 г. Монах Дзякунэн спрятал его от настоятеля, который запрещал говорить о прототипах, называя их «цепями сансары».

```js
const parent = { wisdom: 42 };
const child = Object.create(parent);

child.wisdom; // 42
child.hasOwnProperty('wisdom'); // false
```

Ученик: «Мастер, дитя обладает мудростью. Но говорит, что мудрость — не его.»

Мастер: «Наконец-то честное дитя.»

Ученик: «Но я обращаюсь к child.wisdom — и получаю ответ!»

Мастер снял с шеи чётки и повесил на гвоздь. Под гвоздём — полка. Под полкой — стол. Под столом — пол.

Мастер: «Урони бусину. Куда упадёт?»

Ученик: «На полку. Если нет полки — на стол. Если нет стола — на пол.»

Мастер: «Прототипная цепь. Свойство падает вниз, пока не найдёт опору. Если опоры нет — undefined. Бусина в пустоте.»

Ученик: «А null на конце цепи?»

Мастер: «Конец. Дальше — ничто.»

Бусина падает. Полка. Стол. Пол. Дальше — небо снизу.

С чем ты уйдёшь: Объект владеет только тем, что создал сам. Всё остальное — наследие. Ты знаешь, что в тебе твоё, а что — чужое?

## Коан №16 — «О символе, которого нельзя назвать»

Найден внутри бронзового колокола храма Тисон-ин, 1711 г. Колокол звучал фальшиво сорок лет. Когда его переплавили — нашли свиток. Мастер заметил: «Правильное место для имени, которое нельзя повторить.»

```js
const s = Symbol('koan');
const t = Symbol('koan');

s === t; // false
s.toString(); // 'Symbol(koan)'
```

Ученик: «Мастер, я дал им одинаковое имя. Почему они не равны?»

Мастер указал на двух монахов в одинаковых одеждах.

Мастер: «Оба носят серое. Оба бриты. Оба названы "монах". Они — один человек?»

Ученик: «Нет, но...»

Мастер: «Описание — не сущность. 'koan' — это записка, привязанная к вороту. Для тебя. Не для движка. Каждый Symbol() рождается единственным. Как каждый вдох.»

Ученик: «Зачем нужна уникальность без возможности сравнить?»

Мастер вынул из кармана ключ без надписи.

Мастер: «Этот ключ открывает один замок. На ключе нет имени. На замке нет имени. Но они знают друг друга. Symbol — это ключ. Не имя.»

Два монаха. Одно платье. Разные судьбы.

С чем ты уйдёшь: Symbol создаёт уникальность без имени. В твоём коде есть вещи, которые должны быть уникальными — но ты именуешь их строками и удивляешься коллизиям?

## Коан №21 — «О событии, которое не всплыло»

Фрагмент бересты, найден в стене дзэндо монастыря Нандзэн-дзи при землетрясении 1923 г. Стена треснула снизу вверх. Мастер Кодзан указал на трещину и сказал: «Bubbling.» Затем указал на потолок: «Но кто-то вызвал stopPropagation.»

```js
document.querySelector('.outer').addEventListener('click', () => {
  console.log('внешний');
});

document.querySelector('.inner').addEventListener('click', (e) => {
  e.stopPropagation();
  console.log('внутренний');
});
```

Ученик кликнул по внутреннему. Увидел только: «внутренний»

Ученик: «Мастер, внешний не узнал. Событие остановилось.»

Мастер: «Кто остановил?»

Ученик: «Внутренний элемент.»

Мастер: «Дитя решило, что родитель не должен знать. Ты уверен, что это мудрость?»

Мастер бросил камень в пруд. Круги разошлись к берегам.

Мастер: «Событие рождается в точке удара. Оно всплывает наверх — как круги по воде. stopPropagation — это рука, которая останавливает круги. Берег не узнает об ударе. Если ты остановил событие — ты взял ответственность за всех, кто хотел услышать.»

Камень упал. Круги остановлены. Берег — в неведении.

С чем ты уйдёшь: stopPropagation останавливает не только тех, кого ты знаешь. Кто ещё мог бы услышать это событие — и не услышит теперь?

## Коан №22 — «О петле, которая солгала»

Обнаружен в пепле благовоний на алтаре храма Тэнрю-дзи, 1934 г. Пепел лежал кольцами — пять витков. Послушник прочёл их как for. Мастер Сэки прочёл как closure. Оба были правы. Оба ошибались.

```js
for (var i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 0);
}
// 5 5 5 5 5
```

Ученик: «Мастер, я ждал 0 1 2 3 4. Я получил пять пятёрок.»

Мастер: «Ты получил правду.»

Мастер поставил на стол одну свечу.

Мастер: «var — одна свеча. Одна переменная на весь цикл. Пять колбэков смотрят на неё. Когда они наконец откроют глаза — свеча уже догорела до пяти.»

```js
for (let i = 0; i < 5; i++) {
  setTimeout(() => console.log(i), 0);
}
// 0 1 2 3 4
```

Мастер: «let — пять свечей. Каждая итерация зажигает свою. var существует в функции. let существует в блоке. У каждой итерации своя жизнь.»

Пять колбэков. Одна свеча. Пять пятёрок на полу.

С чем ты уйдёшь: Замыкание закрывает не значение — оно закрывает переменную. Переменная меняется. Ты видишь момент создания или момент исполнения?

## Коан №23 — «О NaN, который не равен себе»

Вырезан на дне медной чаши для подаяний, храм Кэнтё-дзи, 1953 г. Чашу передавали из рук в руки триста лет. Надпись нашли, когда чаша проржавела насквозь. Мастер Асахина сказал: «Наконец-то она стала прозрачной.»

```js
NaN === NaN; // false
```

Мастер ударил ученика палкой.

Мастер: «Почему не спрашиваешь?»

Ученик: «Потому что любой вопрос об этом будет глупым.»

Мастер: «Наконец мудрость.»

Долгое молчание.

Мастер: «NaN — это не значение. Это признание провала. parseInt('камень'). Math.sqrt(-1). Ты попросил число — а мир не смог ответить. NaN — это шрам там, где число должно было быть.»

Ученик: «Как проверить, что это NaN?»

Мастер: «Number.isNaN(). Не голый isNaN() — он лжёт. Только Number.isNaN() говорит правду.»

Вещь без имени. Не равна себе. Есть — и нет.

С чем ты уйдёшь: NaN — единственная сущность в JS, которая отрицает собственное существование. Когда ты в последний раз проверял результат математической операции?

## Коан №24 — «О async/await и иллюзии прямоты»

Найден в заброшенном телеграфном столбе у дороги к храму Мэйдзи-дзингу, 1961 г. Провод оборван. Сообщение не доставлено. Внутри — свиток: «Сообщение ждало. Столб — нет.»

```js
// путь через .then
fetch('/api')
  .then((r) => r.json())
  .then((d) => console.log(d));

// путь через await
const d = await fetch('/api');
const j = await d.json();
console.log(j);
```

Ученик: «Мастер, async/await — это просветление. Код стал прямым. Как синхронный.»

Мастер ударил его палкой трижды.

Мастер: «"Как синхронный" — и есть ловушка.»

Мастер взял лист бумаги, сложил его веером — и развернул обратно.

Мастер: «Гладкий лист. Но следы сгибов остались. await не делает код синхронным. Он прячет промис. Под каждым await — точка разрыва. Функция засыпает. Другой код продолжает работу.»

Ученик: «Но разве это плохо?»

Мастер: «Когда ты забываешь про try/catch — плохо. Когда ты await в цикле последовательно там, где можно параллельно — плохо.»

Прямая дорога. Скрытый мост. Ты идёшь — не зная, что летишь.

С чем ты уйдёшь: async/await — синтаксический сахар над промисами. Сахар не меняет природу. Ты знаешь, что происходит под сахаром?

## Коан №26 — «О destructuring и разделении единого»

Записан тушью на рисовом зерне, найденном в ладони статуи Будды Амиды, храм Котоку-ин, Камакура, 1998 г. Реставратор прочёл под лупой. Заплакал. Уволился. Стал фронтенд-разработчиком.

```js
const universe = { water: { depth: 'бездна', color: 'синий' } };
const { water } = universe;
water.color = 'чёрный';

universe.water.color; // 'чёрный'
```

Ученик: «Мастер, я взял из вселенной только воду.»

Мастер: «Ты взял копию имени. Сама вода — та же.»

Ученик побледнел.

Мастер: «Ты думал, что отделил реку от мира. Но забрал лишь её имя. Тронь её — и мир потемнеет.»

Имя отдельно. Река — та же. Тронул — мир потемнел.

С чем ты уйдёшь: Деструктуризация копирует примитивы и ссылается на объекты. Ты знаешь, что именно держишь в руке — копию или поводок?

## Коан №27 — «О цикле, который не перебирает»

Найден внутри бамбуковой флейты сякухати, принадлежавшей монаху-комусо, деревня Удзи, 2003 г. Флейта не звучала сорок лет. Мастер разрезал бамбук — внутри бумага. На ней — for...in и предупреждение.

```js
const path = ['камень', 'река', 'храм'];

for (const step in path) {
  console.log(step);
}
// '0', '1', '2'
```

Ученик: «Мастер, я получил шаги. Но... это строки?»

Мастер: «Ты получил имена, не вещи. for...in даёт ключи. Для него — это объект с полями '0', '1', '2'.»

Мастер добавил одну строку к прототипу массива.

Мастер: «for...in идёт по всем перечислимым свойствам. Своим — и унаследованным. Ты звал только попутчиков. Пришли и мертвецы предков.»

Флейта молчала сорок лет. Внутри — слова.

С чем ты уйдёшь: for...in и for...of звучат похоже. Первый перебирает ключи, второй — значения. Ты знаешь, что нужно тебе?

## Коан №29 — «О лжи восьми ног»

Извлечён из панциря краба, выловленного в заливе Мацусима, 2017 г. Рыбак отнёс его в храм Дзуйган-дзи. Настоятель прочёл. Побледнел. Сказал: «Краб знал JavaScript.» Краба отпустили.

```js
const falsies = [0, '', null, undefined, NaN, false, -0, 0n];
falsies.filter(Boolean); // []
```

Ученик: «Мастер, я собрал восемь значений. Все — ложь.»

Мастер поднял пустую чашу.

Мастер: «Чаша существует. Чаша пуста. Она ложна?»

Ученик молчал.

Мастер: «Ноль — это число, означающее "ничего". Пустая строка — строка, которая молчит. JavaScript не различает "пустое" и "ложное". Для него молчание — ложь. Но ты — человек. Ты должен различать.»

Восемь крабовых ног. Ни одна не несёт истины. Краб идёт.

С чем ты уйдёшь: Falsy — это не "плохое". Это значения, которые JavaScript считает ложью. Ты тоже так считаешь — или просто доверяешь языку?
