# Коан №8 — «О типе, которого нет»

<Source>Обнаружен в 1991 г. при реставрации храма Бёдо-ин. Пергамент датирован 1053 г. — годом постройки. Архитектор вложил его между балками Зала Феникса. На обороте — typeof null.</Source>

```js
typeof null   // 'object'
```

<Action>Ученик сидел неподвижно три дня. Потом спросил.</Action>

<Action>Мастер не ответил. Он взял пустую чашу и поставил на стол.</Action>

<Master>«Что на столе?»</Master>

<Student>«Чаша.»</Student>

<Master>«Что в чаше?»</Master>

<Student>«Ничего.»</Student>

<Master>«Но ты сказал "чаша". Ты дал ничему форму. Ты поставил пустоту на стол. Теперь пустота занимает место. У неё есть тип. У неё есть адрес в памяти. Это — null.»</Master>

<Student>«Значит, это не ошибка?»</Student>

<Action>Мастер наконец улыбнулся.</Action>

<Master>«Это ошибка. Ей двадцать девять лет. Её нашли в первые десять дней творения. Но миллион храмов построен на этом фундаменте. Исправь — и они рухнут.»</Master>

<Student>«Баг, ставший законом...»</Student>

<Master>«Назови это кармой.»</Master>

<Haiku>Пустая чаша на столе. Внутри — ничего. Тип — object.</Haiku>

<Question>Брендан Айк создал JavaScript за десять дней. typeof null — это не замысел. Это усталость, ставшая стандартом. Сколько таких «typeof null» в твоём собственном коде — решений, принятых наспех, которые теперь слишком дороги для исправления?</Question>
