# Koan №25 — "On the Djinn in the String"

<Source>Discovered on the wall of an underground passage between Kiyomizu-dera and Otowa-no-taki temples, 1977 CE. A spelunker photographed the inscription. The film was exposed. The text was reconstructed from memory — the spelunker turned out to be a Senior Developer.</Source>

<Student>"Master, the string became code! This is power!"</Student>

```js
// WARNING: this is a cautionary example — never execute arbitrary user input
runCode(userInput)
```

<Action>The master stood. Stepped back three paces. Sat again.</Action>

<Master>"Dynamic code execution is a door that opens in both directions. You release the string into the world of execution. But through that same door the world enters your code. Any world. With any intentions."</Master>

<Student>"Then I will not use it."</Student>

<Master>"You will not. But you will build SQL from concatenations. And innerHTML with user input. The djinn takes many forms."</Master>

<Haiku>A string. A wish. The djinn emerged. The door is open in both directions.</Haiku>

<Question>Code is frozen intention. A string is data without intention. Giving data the power of execution is a door in both directions. If any string can become a program — where is the boundary between text and a weapon? And if you never use dynamic execution, but assemble SQL from concatenations — does not the same djinn sit in your bottle, only under a different name?</Question>
