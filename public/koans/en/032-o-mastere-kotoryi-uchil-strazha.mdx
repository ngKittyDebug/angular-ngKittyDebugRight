# Koan №32 — "On the Master Who Taught the Guard and Left"

<Source>Engraved on a silicon wafer found in the foundation of a new data center at the foot of Mount Fuji, 2026 CE. Construction workers mistook it for defective lithography. An engineer read it under a microscope. He wept. He wrote .cursor/rules. He quit. He opened a monastery. In README.md — one line: "The linter sees syntax. The reviewer sees karma."</Source>

<Student>"Master! I configured an AI agent for code review! It checks pull requests!"</Student>

<Master>"What did you tell it?"</Master>

<Student>"'Check the code for errors.'"</Student>

<Master>"And what did it say?"</Master>

<Student>"Said — everything is fine."</Student>

<Source>The master opened the pull request and silently showed the student:</Source>

```js
function processData(data) {
  const result = []
  for (let i = 0; i <= data.length; i++) {
    result.push(data[i])
  }
  return result
}
```

<Student>"No errors. The code works... almost."</Student>

<Action>The master struck with a stick — not the student. The laptop.</Action>

<Master>"Working code is the lowest rung. Absence of errors is not presence of quality. You gave the guard one eye and set him to guard a temple with four gates."</Master>

<Action>The master sat and wrote all day. By evening he handed the student a scroll with a detailed reviewer prompt — about readability, edge cases, off-by-one, performance, intention.</Action>

<Haiku>A guard with one eye. Four gates. Three are open.</Haiku>

<Question>The ideal reviewer agent is not the one who knows all the rules. It is the one who was asked the right question. A prompt is not an instruction. It is the passing of a gaze. If you cannot explain to an agent what bad code is — do you know yourself?</Question>
