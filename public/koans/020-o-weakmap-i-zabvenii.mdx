# Коан №20 — «О WeakMap и забвении»

<Source>Найден в архиве переписки монастыря Сёкоку-дзи, 1891 г. Письмо никогда не было отправлено. Адресат умер. Письмо хранили из уважения. Мастер Сэссю прочёл его через сорок лет и сказал: «WeakRef на человека.»</Source>

```js
const cache = new WeakMap()
let user = { name: 'монах' }

cache.set(user, { lastSeen: Date.now() })

user = null
// объект { name: 'монах' } может быть собран GC
// запись в WeakMap исчезнет вместе с ним
```

<Student>Ученик: «Мастер, Map держит ключ крепко. WeakMap — слабо. Почему?»</Student>

<Master>Мастер: «Map — это архив. Он помнит всё. Даже то, что уже не нужно. Объект в ключе Map никогда не будет собран — пока Map жив.»</Master>

<Student>Ученик: «Утечка памяти?»</Student>

<Master>Мастер: «Привязанность — причина утечек. WeakMap держит слабо. Если объект больше нигде не нужен — WeakMap отпускает без предупреждения. Без траура.»</Master>

<Student>Ученик: «Это жестоко.»</Student>

<Master>Мастер: «Это гигиена.»</Master>

<Haiku>Письмо хранилось. Адресат умер. WeakMap бы отпустил.</Haiku>

<Question>С чем ты уйдёшь: Map хранит сильные ссылки — объект не может быть собран, пока Map о нём помнит. WeakMap держит слабо — и отпускает без предупреждения. Если привязанность — причина утечек памяти, а забвение — лекарство, то что гуманнее: помнить всё и задыхаться, или отпускать и жить легче?</Question>
