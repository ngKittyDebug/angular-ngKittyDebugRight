# Коан №5 — «О живом после смерти»

Свиток из пепла монастыря Рёан-дзи, 1343 г. Монахиня Сэцу-ни нашла его в саду камней, между третьим и четвёртым камнем. Никто не знает, кто положил.

```js
function temple() {
  let candle = 0
  return function() { return ++candle }
}

const pray = temple()
pray() // 1
pray() // 2
pray() // 3
```

Ученик: «Мастер, temple() завершилась. Стек очищен. Контекст мёртв. Но свеча горит.»

Мастер посмотрел на огонь.

Мастер: «Кто сказал тебе, что завершение — это смерть?»

Ученик: «Но... сборщик мусора...»

Мастер: «Сборщик мусора забирает то, о чём забыли. А об этой свече помнят. Внутренняя функция помнит. Пока кто-то помнит — вещь жива.»

Ученик: «Значит замыкание — это память?»

Мастер задул свечу на столе. Дым поднялся.

Мастер: «Замыкание — это призрак, который отказался уйти, потому что его ещё любят.»

Храм сгорел. Монах ушёл. Но в ладонях — тёплый воздух.

С чем ты уйдёшь: Замыкание хранит не значение — а связь с местом, где значение родилось. Если переменная жива только потому, что кто-то на неё ссылается, — чем отличается память JavaScript от человеческой памяти? И что произойдёт, когда последняя ссылка будет потеряна?
