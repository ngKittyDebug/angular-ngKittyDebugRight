# Коан №13 — «О мёртвом коде, который дышит»

<Source>Вырезан на обратной стороне деревянной таблички с именами умерших монахов, храм Эйхэй-дзи, 1603 г. Настоятель Гэнро вырезал его ночью. Утром сделал вид, что табличка была такой всегда.</Source>

```js
function koan() {
  return 'свет'
  console.log('тьма')
}
```

<Student>«Мастер, строка после return — мёртвый код. Она никогда не выполнится.»</Student>

<Master>«Никогда?»</Master>

<Student>«Интерпретатор её не достигнет. Она не существует.»</Student>

<Action>Мастер взял кисть и написал стихотворение на бумаге. Затем перевернул лист и написал ещё одно на обороте.</Action>

<Master>«Прочитай.»</Master>

<Action>Ученик прочёл первое.</Action>

<Master>«А второе?»</Master>

<Student>«Нужно перевернуть лист.»</Student>

<Master>«Но ты знаешь, что оно там. Код после return не выполняется — но занимает байты. Живёт в AST. Парсер его видел. Линтер его оплакивает. Минификатор его похоронит.»</Master>

<Student>«Значит, мёртвый код... существует?»</Student>

<Master>«Он существует для всех, кроме рантайма. Как призрак — всеми видим, никем не признан.»</Master>

<Haiku>Стихотворение на обороте. Рантайм не перевернёт лист.</Haiku>

<Question>Мёртвый код проходит парсинг, занимает место в бандле, читается глазами. Он не выполняется — но влияет. Если код никогда не запустится, но замедлит сборку, запутает разработчика, утяжелит файл — мёртв ли он на самом деле?</Question>
