# Коан №28 — «О запятой, которая родила»

<Source>Соскоблен с лезвия катаны, хранившейся в токонома храма Энгаку-дзи, 2011 г. Меч не вынимали из ножен двести лет. Когда вынули — на стали проступил код. Кузнец был первым программистом. Или последним.</Source>

```js
const x = (1, 2, 3)
console.log(x)  // 3
```

<Student>«Мастер, я передал три числа. Получил одно.»</Student>

<Master>«Ты использовал оператор запятая. Самый тихий оператор в языке. Он вычисляет всё — и возвращает последнее.»</Master>

```js
for (let i = 0, j = 10; i < 5; i++, j--) {
  // i растёт, j убывает
}
```

<Master>«Здесь запятая — слуга. Позволяет двум выражениям жить в одном месте. Безобидна.»</Master>

```js
const result = (doSomething(), getValue())
```

<Master>«А здесь — убийца. Побочный эффект выполнен. Забыт. Только результат остался. Ты спрячешь в запятую вызов API. Мутацию. И никто не заметит — потому что никто не смотрит на запятую.»</Master>

<Haiku>Три монаха вошли. Кто последний — тот и ответ.</Haiku>

<Question>Минификатор любит оператор запятая — он сжимает код. Но код, который легко сжать, легко и спрятать. Если поведение скрыто в синтаксическом сахаре, который никто не читает — существует ли оно для тех, кто поддерживает код после тебя?</Question>
