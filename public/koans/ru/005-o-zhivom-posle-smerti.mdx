# Коан №5 — «О живом после смерти»

<Source>Свиток из пепла монастыря Рёан-дзи, 1343 г. Монахиня Сэцу-ни нашла его в саду камней, между третьим и четвёртым камнем. Никто не знает, кто положил.</Source>

```js
function temple() {
  let candle = 0
  return function() { return ++candle }
}

const pray = temple()
pray() // 1
pray() // 2
pray() // 3
```

<Student>«Мастер, temple() завершилась. Стек очищен. Контекст мёртв. Но свеча горит.»</Student>

<Action>Мастер посмотрел на огонь.</Action>

<Master>«Кто сказал тебе, что завершение — это смерть?»</Master>

<Student>«Но... сборщик мусора...»</Student>

<Master>«Сборщик мусора забирает то, о чём забыли. А об этой свече помнят. Внутренняя функция помнит. Пока кто-то помнит — вещь жива.»</Master>

<Student>«Значит замыкание — это память?»</Student>

<Action>Мастер задул свечу на столе. Дым поднялся.</Action>

<Master>«Замыкание — это призрак, который отказался уйти, потому что его ещё любят.»</Master>

<Haiku>Храм сгорел. Монах ушёл. Но в ладонях — тёплый воздух.</Haiku>

<Question>Замыкание хранит не значение — а связь с местом, где значение родилось. Если переменная жива только потому, что кто-то на неё ссылается, — чем отличается память JavaScript от человеческой памяти? И что произойдёт, когда последняя ссылка будет потеряна?</Question>
