# Коан №19 — «О строке, которая не строка»

<Source>Записан на обёртке от благовоний в монастыре Кофуку-дзи, 1850 г. Благовония сгорели. Обёртка — нет. Мастер сказал: «Содержимое исчезло. Форма осталась. Это шаблонный литерал.»</Source>

```js
const name = 'монах'
const greeting = `Приветствую, ${name}`

// Но также:
const html = String.raw`<div class="${name}"></div>`
```

<Student>«Мастер, это просто удобная строка с подстановкой.»</Student>

<Master>«А если я скажу тебе, что строка — это вызов функции?»</Master>

<Student>«Строка — это функция?»</Student>

```js
function zen(strings, ...values) {
  return strings.raw[0] + values[0].toUpperCase()
}

zen`Привет, ${'мир'}`  // 'Привет, МИР'
```

<Master>«Tagged template. Функция перед обратными кавычками — это тег. Она получает части строки и подстановки отдельно. html\`...\`, css\`...\`, gql\`...\` — это языки внутри языка.»</Master>

<Student>«Строка стала языком...»</Student>

<Haiku>Обёртка пуста. Но форма — язык.</Haiku>

<Question>Tagged templates превращают строковый литерал в вызов функции. Строка перестаёт быть данными и становится языком внутри языка. Если обратные кавычки могут создать встроенный DSL — где граница между строкой, кодом и языком?</Question>
