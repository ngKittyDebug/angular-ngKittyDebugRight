# Коан №13 — «О мёртвом коде, который дышит»

Вырезан на обратной стороне деревянной таблички с именами умерших монахов, храм Эйхэй-дзи, 1603 г. Настоятель Гэнро вырезал его ночью. Утром сделал вид, что табличка была такой всегда.

```js
function koan() {
  return 'свет'
  console.log('тьма')
}
```

Ученик: «Мастер, строка после return — мёртвый код. Она никогда не выполнится.»

Мастер: «Никогда?»

Ученик: «Интерпретатор её не достигнет. Она не существует.»

Мастер взял кисть и написал стихотворение на бумаге. Затем перевернул лист и написал ещё одно на обороте.

Мастер: «Прочитай.»

Ученик прочёл первое.

Мастер: «А второе?»

Ученик: «Нужно перевернуть лист.»

Мастер: «Но ты знаешь, что оно там. Код после return не выполняется — но занимает байты. Живёт в AST. Парсер его видел. Линтер его оплакивает. Минификатор его похоронит.»

Ученик: «Значит, мёртвый код... существует?»

Мастер: «Он существует для всех, кроме рантайма. Как призрак — всеми видим, никем не признан.»

Стихотворение на обороте. Рантайм не перевернёт лист.

С чем ты уйдёшь: Мёртвый код проходит парсинг, занимает место в бандле, читается глазами. Он не выполняется — но влияет. Если код никогда не запустится, но замедлит сборку, запутает разработчика, утяжелит файл — мёртв ли он на самом деле?
