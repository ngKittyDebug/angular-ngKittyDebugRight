# Коан №15 — «О микрозадаче и макрозадаче»

<Source>Найден в расписании дежурств монастыря Тофуку-дзи, 1720 г. Между строк о уборке — колонка с приоритетами задач. Мастер Хакуин добавил её красными чернилами. Монахи решили, что это украшение.</Source>

```js
console.log('1')
setTimeout(() => console.log('2'), 0)
Promise.resolve().then(() => console.log('3'))
console.log('4')
// 1, 4, 3, 2
```

<Student>Ученик: «Мастер, промис выполнился быстрее таймера. Но оба были готовы одновременно.»</Student>

<Master>Мастер: «Готовы — не значит одинаковы. Есть очередь срочных дел. И есть очередь обычных.»</Master>

<Student>Ученик: «Микрозадачи и макрозадачи?»</Student>

<Master>Мастер: «После каждой задачи из обычной очереди — Event Loop опустошает очередь срочных полностью. Промис попал в срочную. setTimeout — в обычную. Срочное всегда раньше обычного.»</Master>

<Student>Ученик: «Тогда промисы всегда быстрее?»</Student>

<Master>Мастер: «Промис, порождающий промис, порождающий промис — может заблокировать рендеринг навсегда. Приоритет без меры становится тиранией.»</Master>

<Haiku>Срочное — первым. Но очередь срочных — бездонна.</Haiku>

<Question>С чем ты уйдёшь: Микрозадачи приоритетнее — но приоритет без меры становится тиранией. Если промис порождает промис, а тот — следующий, Event Loop никогда не доберётся до рендеринга, до пользователя. Что опаснее — задача, которая ждёт слишком долго, или задача, которая не умеет остановиться?</Question>
