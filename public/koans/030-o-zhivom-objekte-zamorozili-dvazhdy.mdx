# Коан №30 — «О живом объекте, который заморозили дважды»

Записан инеем на оконном стекле серверной комнаты монастыря Коясан, зима 2024 г. К утру растаял. Админ успел сделать скриншот. Скриншот хранится в /dev/null. Мастер сказал: «Правильное место.»

```js
const config = Object.freeze({
  db: { host: 'localhost', port: 5432 },
  cache: true
})

config.cache = false      // тихо игнорируется
config.db.host = 'prod'   // работает!
```

Мастер вынес чашу воды на мороз. Поверхность замёрзла. Он проткнул лёд пальцем — внизу вода.

Мастер: «Object.freeze — первый лёд. Поверхность. Ключи верхнего уровня — неприкосновенны. Но вложенный объект — это ссылка. Ты заморозил указатель — не комнату, в которую он указывает.»

Ученик: «Тогда я заморожу глубже!»

```js
function deepFreeze(obj) {
  Object.values(obj).forEach(val => {
    if (typeof val === 'object') deepFreeze(val)
  })
  return Object.freeze(obj)
}
```

Мастер: «Хорошо. А теперь — циклическая ссылка.»

Ученик: «Бесконечная рекурсия...»

Мастер: «Ты хотел заморозить мир. Но мир ссылается сам на себя.»

Лёд сверху. Вода внутри. Мороз не достаёт до дна.

С чем ты уйдёшь: Object.freeze замораживает поверхность. Если неизменность только сверху — это неизменность или иллюзия контроля? И сколько слоёв нужно заморозить, чтобы система стала действительно предсказуемой?
