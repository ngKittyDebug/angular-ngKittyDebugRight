# Sprint 4: Sanctum — целая фича с помощью ИИ — 2026-06-26

## What was done

На этой неделе я сделала целую фичу — **Sanctum веток** (`/sanctum`): страница, где можно «осудить» имя git-ветки через ритуал с литанией, вердиктом (святость, судьба merge, знамения, пророчество) и цифровым священником.

Главное отличие от прошлых задач: **большую часть работы делала в Cursor с ИИ**, а не руками с нуля. Я не просто «попросила написать код» — я использовала агента как напарника по всему циклу: ревью, рефакторинг, дизайн, тесты, коммиты и PR.

Как это выглядело по шагам:

1. **Старт с ревью.** Сначала попросила ИИ пройтись по уже набросанному sanctum-коду и сказать, что сломано / раздуто. Он нашёл kebab-case баг, лишние файлы (~88 штук), дублирование логики. Я согласилась на переписывание — получилось компактнее (~50 файлов), с нормальным разделением domain / services / facade / ui.

2. **Domain без Angular.** Хелперы вроде `analyzeBranch`, конфиги паттернов веток, omens, litany — чистые функции + тесты. ИИ хорошо генерит такую «математику», я правила тексты пророчеств и пороги score под наш вайб.

3. **Ритуал на rxResource.** Таймлайн литании через `timer` + `merge`, оркестрация в `SanctumRitualService`. ИИ предложил effect для звуков — **не сработало** (звуков не было), потом мы вернули `tap` в stream и `prime()` AudioContext на клик submit. Урок: слепо не мержить рефакторинг от ИИ.

4. **Digital Priest.** Несколько итераций дизайна: «бабка» из git, смесь с моим фото, более церковно-digital стиль, фикс рта при rage (чтобы не «улыбался» когда ругается), фикс скачка layout при речи. Тут ИИ сильнее всего как «быстрый дизайнер» — я описывала, что не нравится, он крутил SCSS/SVG.

5. **Цитаты священника из Firestore.** Resource API, пулы цитат по mood, группировка — опять же ИИ набросал структуру, я проверяла по project conventions (route-scoped providers, `@features/*`).

6. **Звук.** Web Audio: орган, колокол, reverb «собор». Громкость и «церковность» — итерации в чате («тише», «погромче», «поцерковнее»). Звучит смешно, но так и работало.

7. **Git hygiene.** Попросила разбить изменения на логичные коммиты, запушить, обновить описание PR на русском. Conventional commits, pre-push тесты — всё прошло.

PR в `develop`: [#259 feat(sanctum): add branch judgment ritual page](https://github.com/ngKittyDebug/angular-ngKittyDebugRight/pull/259).

## Problems

- **ИИ переусложняет или ломает рабочее.** Effect вместо `tap` для звуков — классический пример: красиво в теории, в браузере тишина из-за autopolicy и статуса rxResource.
- **Дизайн «на один промпт» редко попадает.** Священник — штук 5–6 итераций, пока не стало ок.
- **Контекст длинных сессий.** Иногда агент «забывает» что уже делали (navigation, hover в light theme) — приходилось напоминать или кидать скрин.
- **Всё равно нужно понимать код.** Без Angular 22 / signals / rxResource я бы не поняла, где ИИ накосячил.

## Solutions

- Дробила задачи: «сначала ревью», «потом только domain», «отдельно звук», «отдельно коммиты» — так проще контролировать diff.
- После каждого крупного куска — `pnpm test` / typecheck (часто просила ИИ самому прогнать).
- Для UI — скриншоты в чат, не абстрактные «сделай красиво».
- Когда звук пропал — описала симптом, агент нашёл regression в коммите `9444a34` и откатил подход.

## What I learned

- **ИИ реально может вытащить фичу end-to-end**, если ты задаёшь архитектурные рамки (facade pattern, pure helpers, route providers) и не принимаешь первый ответ.
- **Cursor + agent transcripts** — удобно продолжать сессию после summary, но лучше самой помнить «что уже в PR».
- `rxResource` + Observable-таймлайн — норм для «театрального» UX (литания по строкам).
- Web Audio: без `AudioContext.resume()` на user gesture звука не будет, как ни крути reverb.
- Разбиение на мелкие коммиты через ИИ экономит время, но сообщения всё равно стоит прочитать.
