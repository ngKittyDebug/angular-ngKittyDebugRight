### Гибкая структура через @if

Каждый элемент карточки опциональный - появляется только если передан соответствующий `input`:

- иконка
- заголовок
- подзаголовок
- описание
- бейдж
- кнопка

```typescript
readonly icon        = input<string>();
readonly title       = input<string>();
readonly subtitle    = input<string>();
readonly decryption  = input<string>();
readonly badge       = input<string>();
readonly buttonLabel = input<string>();
readonly isClickable = input<boolean>(false);
```

```html
@if (title()) {
<h2 class="title">{{ title() }}</h2>
} @if (buttonLabel()) {
<button (click)="onButtonClick($event)">{{ buttonLabel() }}</button>
}
```

---

### Почему input(), а не сервис?

Данные передаются через `input()` - это связь родитель → ребёнок, самый подходящий способ. Если бы компоненты не были связаны иерархически - понадобился бы сервис.

```html
<!-- родитель передаёт данные через атрибуты -->
<ngKitty-card
  [icon]="'@tui.skull'"
  [title]="'Legacy Code'"
  [subtitle]="'Ancient systems'"
  [decryption]="'Respect the ancient wisdom.'"
  [badge]="'100%'"
  [buttonLabel]="'Explore'"
  [isClickable]="true"
  (buttonClick)="onButtonClick($event)"
  (cardClicked)="onCardClick()"
/>
```

---

### output() — уведомления наружу

Карточка не знает что произойдёт после нажатия - она только сообщает о событии через `emit()`. Логику обрабатывает родитель.

```typescript
readonly buttonClick = output<string>();
readonly cardClicked = output<void>();

onButtonClick(event: MouseEvent): void {
  event.stopPropagation();
  this.buttonClick.emit(this.title() ?? '');
}

onCardClick(): void {
  if (this.isClickable()) {
    this.cardClicked.emit();
  }
}
```

`stopPropagation()` нужен потому что клик по кнопке "всплывает" к родительскому элементу (карточке). Без него сработают оба обработчика сразу.

### host — убираем лишний div

Тимлид Даша предложила посмотреть на `host` в декораторе компонента.

**Без host** в шаблоне был лишний `<div class="card">` — просто оболочка для стилей и обработчиков.

**С host** этот div не нужен — все атрибуты и обработчики событий вешаются прямо на тег компонента (`<ngKitty-card>`), который и является host-элементом в DOM.

```typescript
@Component({
  selector: 'ngKitty-card',
  changeDetection: ChangeDetectionStrategy.OnPush,
  host: {
    '[attr.tabindex]': 'isClickable() ? 0 : null', // фокусируемый если кликабельный
    '(click)':         'onCardClick()',
    '(keydown.enter)': 'onCardClick()',             // доступность с клавиатуры
  },
})
```

Стили тоже переехали — с `.card {}` на `:host {}` в scss:

```scss
:host {
  display: flex;
  flex-direction: column;
  align-items: center;
  gap: 1rem;
  padding: 1rem;
  border: 1px solid var(--tui-border-normal);
  background: var(--tui-background-elevation-2);
}
```

`tabindex="0"` делает элемент фокусируемым с клавиатуры (Tab-навигация). Если `isClickable = false` — атрибут убирается совсем (`null`).
